🤖 AI CODING TOOLS: CLAUDE CODE vs GEMINI CLI vs ANTIGRAVITY
Hướng dẫn dành cho người mới bắt đầu  ·  Data Analyst Team  ·  2026

PHẦN 1 — 3 CÔNG CỤ AI LÀ GÌ? TẠI SAO LẠI SO SÁNH CHÚNG?
Trong thế giới phát triển phần mềm, ba trợ lý AI dòng lệnh (CLI — Command Line Interface) đang nổi bật nhất hiện nay là Claude Code, Gemini CLI và AntiGravity. Cả ba đều hỗ trợ lập trình viên (developer) và nhà phân tích dữ liệu (data analyst — DA) làm việc hiệu quả hơn — nhưng theo những cách khác nhau. Phần này giải thích ngắn gọn bản chất của từng công cụ.

🟢  Claude Code | 🔵  Gemini CLI | 🟣  AntiGravity
Bản chất: Agent tự hành (Autonomous Agent) — giao đề bài, nó tự thực thi.  Nhà phát triển: Anthropic  Ví von: Như trợ lý thông minh cần “sổ tay nội quy” (file CLAUDE.md). Muốn nó làm đúng ý bạn thì bạn phải tự viết, tự cập nhật file đó.  Điểm mạnh: Lập luận độc lập (autonomous reasoning) cực tốt. Ít cần con người can thiệp giữa chừng.  Điểm yếu: Bộ nhớ (memory) phụ thuộc hoàn toàn vào bạn tự ghi chép. | Bản chất: Trợ lý CLI gọn nhẹ, tập trung vào scripting (viết script tự động) & API.  Nhà phát triển: Google  Ví von: Chuyên gia xử lý tức thì (Just-in-Time). Vào thư mục nào thì đọc file GEMINI.md của thư mục đó, không nhớ lịch sử nhiều.  Điểm mạnh: Nhanh gọn, phù hợp tác vụ đơn giản, tích hợp tốt với Google Cloud.  Điểm yếu: Không có hệ thống nhớ dài hạn (long-term memory) chuyên biệt. | Bản chất: Autonomous Agent với Hybrid Interface (giao diện kết hợp sidebar UI + backend agentic).  Nhà phát triển: Google DeepMind  Ví von: “Đồng nghiệp siêu phàm” có bộ não riêng — tự học, tự nhớ, tự lên kế hoạch nhiều bước và hiển thị kết quả đẹp như slide báo cáo.  Điểm mạnh: Knowledge Items (KIs — kho kiến thức tự học); Artifacts (tài liệu xuất đẹp); Browser Subagent (agent con tự điều khiển web).  Điểm yếu: Cần kết nối internet; phụ thuộc vào model được tích hợp sẵn (built-in model).

Tiêu Chí | Claude Code 🟢 | Gemini CLI 🔵 | AntiGravity 🟣
Nhà phát triển | Anthropic | Google | Google DeepMind
Giao diện chính | Terminal (CLI — giao diện dòng lệnh) | Terminal (CLI) | Hybrid: Sidebar UI + Agentic Backend (nền tảng agentic)

Bộ nhớ dài hạn (long-term memory) | 🔶 Thủ công — phải tự ghi vào CLAUDE.md | ❌ Không có hệ thống chuyên biệt | ✅ Tự động — Knowledge Items tự chắt lọc từ hội thoại cũ
Lịch sử hội thoại (conversation history) | ✅ Có cơ chế claude history | ❌ Chỉ lưu session ngắn (short session) | ✅ Đầy đủ: brain/<id>/overview.txt (Full Transcript)
Conversation IDs (mã định danh phiên làm việc) | ✅ Có ID để tiếp tục (Resume) | ❌ Không có | ✅ UUID riêng cho mỗi session

Điều phối quy trình (workflow orchestration) | Bash, đọc/ghi file trực tiếp | Scripting & API Call | Tool chuyên sâu: multi_replace, run_command (async tracking — theo dõi bất đồng bộ)
Hỗ trợ MCP (Model Context Protocol) | ✅ Rất mạnh — Anthropic là tác giả chuẩn MCP | 🔶 Hạn chế / Đang thử nghiệm | ✅ Tích hợp qua mcp_config.json
Hiển thị kết quả (Artifacts) | ❌ Text thô trên Terminal | ❌ Terminal output | ✅ Markdown đẹp, Mermaid diagrams, Carousel slides, Image AI
Điều khiển trình duyệt (browser automation) | ❌ Không tích hợp sẵn | ❌ Không có | ✅ Browser Subagent: click, gõ, cuộn, ghi hình video .webp

PHẦN 3 — CLAUDE CODE vs GEMINI CODE ASSIST: HAI LOẠI KHÁC NHAU
⚠️ Lưu ý quan trọng: Gemini CLI (dùng terminal — dòng lệnh) khác hoàn toàn với Gemini Code Assist (tích hợp vào IDE — Môi trường phát triển tích hợp như VS Code). Bảng dưới giúp phân biệt rõ vai trò của từng công cụ để tránh nhầm lẫn khi chọn đúng tool cho đúng nhu cầu:
Tiêu chí | Claude Code (Agent tự hành) | Gemini Code Assist (AI tích hợp IDE)
Bản chất | Agent (độc lập) — tự chạy terminal, đọc/ghi file, kiểm thử (testing), sửa lỗi thay bạn. | Trợ lý tích hợp sâu trong IDE — gợi ý, autocomplete (hoàn thành code tự động), giải thích code theo ngữ cảnh (context-aware).
Cách tương tác | Qua Terminal hoặc bảng điều khiển Agent. Giao việc xong chờ kết quả. | Side-panel (bảng bên cạnh) trong IDE hoặc autocomplete khi gõ code. Liên tục đồng hành.
Phù hợp cho task gì? | "Viết unit test (kiểm thử đơn vị) cho module X, chạy thử, nếu lỗi thì sửa và commit lên Git." | "Doạn code này nghĩa là gì?", "Gợi ý cách tối ưu query SQL này."
Điểm mạnh nhất | Reasoning (lập luận) & thực thi (execution) độc lập. Ít cần con người can thiệp giữa chừng. | Context Window (cửa sổ ngữ cảnh) khổng lồ — đọc hàng triệu dòng code. Kết nối Google Cloud/Firebase tốt.
Đối tượng phù hợp | Dev/DA muốn "giao khoán" — đưa đề bài, nhận kết quả (Artifacts). | Dev/DA muốn “đồng hành” liên tục, hỏi nhanh và tra cứu tài liệu.

PHẦN 4 — 4 TÍNH NĂNG "HAY HO" CỦA ANTIGRAVITY MÀ NGƯỜI MỚI CẦN BIẾT
1. Kiến trúc Autonomous Agent (Tác tử Tự trị)
AntiGravity không đơn giản là một chatbox (hộp chat thông thường). Khi bạn giao một task phức tạp, nó tự động thực hiện toàn bộ vòng lặp sau:
▸  Lên kế hoạch thực hiện từng bước (Planning — lập kế hoạch)
▸  Chạy lệnh terminal (ví dụ: npx, pip install, git commit)
▸  Đọc/viết file code trực tiếp vào project (file I/O — nhập/xuất tệp)
▸  Tạo ảnh minh họa AI (Generate Image) thay vì để placeholder (ô trống chờ) trống
▸  Tự kiểm tra kết quả và báo lại — không cần bạn can thiệp giữa chừng.

2. Hệ thống "Bộ não" — Knowledge Items (KIs — Kho Kiến Thức)
Đây là thứ khiến AntiGravity khác biệt hoàn toàn. Thay vì nhớ bằng file tĩnh (static file — như CLAUDE.md phải tự viết tay), AntiGravity tự động chắt lọc (distill) bài học từ các phiên làm việc trước, lưu vào:
  📁  C:\Users\<you>\.gemini\antigravity\knowledge\
▸  Hỏi về lỗi quen thuộc? → Nó tự tra KI xem trước đó sửa thế nào.
▸  Bạn thường dùng naming convention (quy ước đặt tên) nào? → Nó tự nhớ và áp dụng nhất quán.
▸  Không cần "dạy lại" mỗi lần mở hội thoại (session) mới.

3. Thao tác Trình duyệt — Browser Subagent (Agent Con Điều Khiển Trình Duyệt)
AntiGravity có thể kích hoạt một “phụ tá ảo” (subagent) mở trình duyệt ngầm (headless browser) để:
▸  Truy cập trang web, đọc nội dung, cuộn trang (scroll)
▸  Thực hiện click và nhập dữ liệu tự động (web automation)
▸  Ghi lại toàn bộ thao tác thành video .webp hoặc ảnh screenshot (ảnh chụp màn hình)
▸  Lưu kết quả vào folder Artifacts để báo cáo cho bạn
🎯 Ứng dụng cho DA team: Tự động crawl (thu thập) dữ liệu, kiểm tra giao diện dashboard (bảng điều khiển), chụp màn hình báo cáo.

4. Artifacts — Hệ thống Hiển thị Báo cáo Chuyên nghiệp
Thay vì đọc 500 dòng text trắng trên nền đen, AntiGravity xuất kết quả ra file Artifacts (.md — Markdown) với:
▸  Bảng biểu (Tables) và biểu đồ kiến trúc (Mermaid diagrams — sơ đồ dạng code)
▸  Carousel slides (trang trình chiếu dạng băng chuyền) — nhấn mũi tên để xem từng bước phân tích
▸  Code diff (so sánh trước/sau chỉnh sửa) có màu highlight (tô sáng)
▸  File link có thể click trực tiếp tới đoạn code liên quan (deep linking — liên kết sâu)
✅ Kết quả: Hội thoại chat ngắn gọn, báo cáo tài liệu rõ ràng — rất phù hợp để bàn giao (handover) với team.

PHẦN 5 — TIPS THỰC CHIẾN CHO DA TEAM
💡  Prompt chất lượng cao:  Khi dùng AntiGravity, thêm từ khóa chất lượng vào prompt (câu lệnh bạn gõ): "Premium design", "Dynamic aesthetics", "Modern typography" → Kết quả UI/dashboard cải thiện đáng kể so với prompt thông thường.
💡  Tận dụng hệ thống KIs:  Sau mỗi phiên phân tích quan trọng, hỏi AntiGravity: "Hãy lưu pattern query (mẫu câu truy vấn) này vào Knowledge Items." Lần sau nó sẽ tự áp dụng mà không cần nhắc lại.
💡  File hướng dẫn _GEMINI.md trong project:  Đặt file _GEMINI.md ở thư mục gốc (root directory) project của team với mô tả: cấu trúc thư mục, naming convention (quy ước đặt tên), tool thường dùng. AntiGravity sẽ đọc file này ngay khi mở project và làm việc đúng context (ngữ cảnh).
💡  Không dùng BYOK (Bring Your Own Key — tự mang API key):  AntiGravity hiện không hỗ trợ đổi model (mô hình AI) tùy ý vì các khả năng Agentic (tool-calling, browser subagent...) được Google DeepMind huấn luyện riêng trên các model tích hợp sẵn (built-in). Bạn vẫn có thể viết code gọi API bất kỳ — nhưng AntiGravity (agent) chỉ chạy trên bộ model được tích hợp.

PHẦN 6 — MCP — MODEL CONTEXT PROTOCOL LÀ GÌ?
🔌 MCP (Model Context Protocol — Giao thức Ngữ cảnh Mô hình) giống như ổ cắm điện chuẩn quốc tế. Nhà sản xuất thiết bị (quạt, tủ lạnh, laptop...) chỉ cần làm cái phích cắm đúng chuẩn — thì cắm vào ổ điện nước nào cũng chạy được. Không cần hỏi ổ điện đó của hãng nào.
MCP là chuẩn giao tiếp mở (open standard) do Anthropic tạo ra, cho phép AI Agent (tác tử AI) “cắm vào” bất kỳ công cụ/dịch vụ bên ngoài nào — miễn là công cụ đó “chìa ra” một MCP Server (máy chủ MCP).
🛠️ Cơ chế hoạt động — 3 thành phần
Thành phần | Tên kỹ thuật | Vai trò | Ví dụ
Bên dùng tool | MCP Client | AI Agent đặt câu hỏi “Mày có thể làm gì?” và gọi tool | Claude, AntiGravity, Gemini CLI
Bên cung cấp tool | MCP Server | Công cụ bên ngoài tự “đăng ký” (expose) khả năng của mình | BigQuery, GitHub, Notion, Slack
Ngôn ngữ chung | MCP Protocol | Định dạng JSON chuẩn để 2 bên hiểu nhau (interoperability — khả năng tương thích) | JSON-RPC qua stdio/HTTP

🌍 Ví dụ thực tế — Tool nào đã có MCP Server?
Công cụ | MCP Server cung cấp gì? | Ứng dụng cho DA Team
PostgreSQL / BigQuery | query(), list_tables() | AI tự viết & chạy SQL (Structured Query Language — ngôn ngữ truy vấn cơ sở dữ liệu), trả về kết quả
GitHub | create_pr(), read_file(), commit() | AI tự commit code, tạo Pull Request (PR — yêu cầu gộp code)
Slack | send_message(), read_channel() | AI tự post kết quả phân tích lên channel (kênh nhóm)
Notion | create_page(), search_docs() | AI tự tạo báo cáo, điền vào trang Notion
Brave Search | web_search() | AI tự tìm kiếm web, không cần bạn copy-paste
Filesystem | read_file(), write_file() | AI đọc/ghi file CSV, Excel trực tiếp (file system access — truy cập hệ thống tệp)
Puppeteer | click(), screenshot() | AI điều khiển trình duyệt (web browser automation — tương tự Browser Subagent của AntiGravity)

⚖️ So sánh mức độ hỗ trợ MCP giữa 3 công cụ
Tiêu chí | Claude Code | Gemini CLI | AntiGravity
Có hỗ trợ MCP không? | ✅ Có — rất đầy đủ | 🔶 Đang phát triển (experimental — thử nghiệm) | ✅ Có — qua mcp_config.json
File config (tệp cấu hình) | .claude/mcp.json | Tuỳ version (phiên bản) | mcp_config.json
Ai tạo ra chuẩn MCP? | Anthropic (chính chủ — original author) | Google (sau đó adopt — áp dụng theo) | Google DeepMind (adopt)
Thêm tool mới dễ không? | ✅ Rất dễ — ecosystem (hệ sinh thái) lớn nhất | 🔶 Ít plugin (tiện ích mở rộng) hơn | ✅ Dễ — viết hoặc tải MCP Server từ cộng đồng

📌 Vì Anthropic là tác giả của chuẩn MCP, nên Claude Code có lợi thế về ecosystem (hệ sinh thái) — nhiều tool bên thứ 3 (third-party) viết MCP Server cho Claude trước, rồi mới porting (chuyển đổi) sang công cụ khác.

Ví dụ thực tế: Nếu team có database BigQuery và cài MCP Server cho BigQuery, bạn có thể nói với AntiGravity:
“Lấy dữ liệu doanh số tháng 3 từ BigQuery, vẽ biểu đồ trend (xu hướng) và lưu báo cáo vào Notion.”
→ AntiGravity sẽ tự gọi BigQuery qua MCP, xử lý data (dữ liệu), rồi gọi Notion qua MCP để tạo trang báo cáo — không cần bạn làm thủ công bước nào.
